Database Schema

The database covers six clouds, each with the table prefix aws, gcp, azure, stackit, ovh or oracle. Prices are Linux on-demand rates in USD per hour from one low-price reference region per cloud (AWS: us-east-1, GCP: us-central1, Azure: eastus, STACKIT: eu01, OVHcloud: standard region, Oracle: uniform list price).

Table / ViewContents
<cloud>_allEvery instance type of that cloud. All six tables share an identical 27-column schema, so clouds are directly comparable, e.g. select * from aws_all union all select * from gcp_all.
<cloud>Comparable slice: current-generation, priced, non-burstable, non-accelerator, non-metal instances (non-standard instances dropped).
<cloud>_familyOne representative instance per family.
<cloud>_accelGPU/accelerator instances.
<cloud>_burstBurstable / shared-core instances (vcpus_base < vcpus).

Columns of <cloud>_all

Values a cloud does not publish are NULL.
